### 2.4 ناسازگاری WBS / Outline (Data Quality)

برخی Taskها بین `WBS Code` و `Outline Number` ناسازگاری دارند؛ مثلاً:

| uid | WBS | Outline | نام |
|-----|-----|---------|-----|
| 578 | `2.1` | `1.5.2` | اجرای فاز یک پروژه ۲۵ هکتاری |
| 579 | `2.1.1` | `1.5.3` | اجرا سوله |
| 580 | `2.1.2` | `1.5.4` | تهیه اسناد مناقصه اجرای سوله |

→ Importer نباید حدس بزند؛ این ردیف‌ها باید در بخش **Ambiguous Mapping** برای تأیید
دستی Editor نمایش داده شوند (رجوع به `import-validation.md`). مرجع اصلی ساختار سلسله‌مراتب
**Outline Level/Number** است نه WBS Code خام.

## 3. رابطهٔ Excel ⟷ MPP (خلاصه)

- **Excel** = منبع اصلی Taxonomy هفت‌فازی و ساختار کلان (۷ فاز، ۲۴ Break1، ۱۴۲ فعالیت).
- **MPP** = برنامهٔ فیزیکی تفصیلی و Scheduling، منحصراً برای **فاز ۵** (پروژهٔ ۲۵ هکتاری).
- کلید تطبیق: Outline Number → WBS → عنوان Normalize‌شده → تاریخ‌ها → Manual Mapping.
- Precedence پیش‌فرض در `data-mapping.md`.

جزئیات نگاشت میدانی در `data-mapping.md`، قواعد محاسبات در `calculation-rules.md`،
و قواعد اعتبارسنجی Import در `import-validation.md` آمده است.
