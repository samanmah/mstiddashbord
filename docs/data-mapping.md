# نگاشت دادهٔ Excel به دیتابیس (Data Mapping)

Importer بر اساس **تشخیص عنوان ستون‌ها** (نه شماره ثابت سطر) کار می‌کند تا فایل‌هایی با تعداد فعالیت/ریسک بیشتر نیز قابل ورود باشند. نام شیت‌ها و عنوان ستون‌ها پیش از تطبیق Normalize می‌شوند.

## قواعد Normalize متن

- حذف فاصله‌های اضافی ابتدا/انتها و فشرده‌سازی فاصله‌های متوالی.
- تبدیل نیم‌فاصله (`\u200c`) و Zero-Width (`\u200b`،`\u200f`،`\u200e`،`\uFEFF`) به فاصله/حذف.
- یکسان‌سازی «ي/ی» عربی→فارسی و «ك/ک» عربی→فارسی.
- تبدیل اعداد فارسی `۰۱۲۳۴۵۶۷۸۹` و عربی `٠١٢٣٤٥٦٧٨٩` به لاتین.
- حذف `%`، مدیریت `,` هزارگان.
- تبدیل `None`، `null`، رشته خالی و `-` به `null`.

## نگاشت نام شیت‌ها

## ۲. شیت «پیشرفت ماهیانه»

| ستون Excel | فیلد مدل | نوع |
|------------|----------|-----|
| ماه | `monthLabel` + `jalaliYear` + `jalaliMonth` + `sortOrder` | Parse از `«نام‌ماه (سال)»` |
| برنامه (%) | `plannedPercent` | number |
| واقعی (%) | `actualPercent` | number? (خالی→null) |
| انحراف (%) | — | **نادیده** (محاسبه‌ای: actual - planned) |

نگاشت نام ماه جلالی → شماره ماه: فروردین=1 … اسفند=12. `date` اولین روز ماه جلالی ذخیره می‌شود.

## نگاشت Enumها (فارسی → دیتابیس → نمایش UI)

### Probability
| فارسی Excel | Enum | نمایش UI |
|-------------|------|----------|
| پایین / کم | `LOW` | پایین |
| متوسط | `MEDIUM` | متوسط |
| بالا / زیاد | `HIGH` | بالا |

### RiskLevel
| فارسی Excel | Enum | نمایش UI |
|-------------|------|----------|
| پایین / کم | `LOW` | پایین |
| متوسط | `MEDIUM` | متوسط |
| بالا / زیاد | `HIGH` | بالا |

### ActivityStatus (محاسبه‌ای)
| Enum | شرط | نمایش UI | رنگ |
|------|-----|----------|-----|
| `GOOD` | ratio ≥ 0.90 | خوب | سبز |
| `AVERAGE` | 0.70 ≤ ratio < 0.90 | متوسط | نارنجی |
| `WEAK` | ratio < 0.70 | ضعیف | قرمز |
| `UNKNOWN` | planned = 0 | نامشخص | خاکستری |

نگاشت وضعیت فارسی فایل → Enum (برای `statusOverride`): خوب→GOOD، متوسط→AVERAGE، ضعیف→WEAK، نامشخص→UNKNOWN.

### DecisionStatus
| فارسی Excel | Enum | نمایش UI | رنگ |
|-------------|------|----------|-----|
| جدید | `NEW` | جدید | آبی |
| در حال اجرا | `IN_PROGRESS` | در حال اجرا | نارنجی |
| در انتظار گزارش | `WAITING_FOR_REPORT` | در انتظار گزارش | بنفش |
| انجام شد | `DONE` | انجام شد | سبز |
| سایر | `OTHER` | سایر | خاکستری |

## قواعد تاریخ جلالی

- فرمت ورودی `YYYY/MM/DD` (اعداد فارسی/عربی/لاتین پذیرفته می‌شود).
- اعتبارسنجی تعداد روز ماه‌ها (۱..۶ = ۳۱ روز، ۷..۱۱ = ۳۰ روز، ۱۲ = ۲۹ یا ۳۰ در کبیسه).
- تبدیل جلالی↔میلادی با الگوریتم دقیق (بدون وابستگی به Locale سیستم‌عامل).
- ذخیره در Postgres به‌صورت `Date` (UTC-safe، ساعت ۱۲ ظهر برای جلوگیری از خطای Timezone).
- Timezone نمایش: `Asia/Tehran`.
